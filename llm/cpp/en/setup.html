Setup and Build

Requirements:

  • Xcode 14.2 and higher

Requirements:

Building the sample

Move to the cpp folder, then run the command corresponding to your OS as described below.

Windows

cl gemma4.cpp ailia_llm.lib

macOS

clang++ -o gemma4 gemma4.cpp libailia_llm.dylib -Wl,-rpath,./ -std=c++17

Linux

export LD_LIBRARY_PATH=./
g++ -o gemma4 gemma4.cpp libailia_llm.so

Running the sample

Use the launcher script (gemma4.sh on macOS/Linux or gemma4.bat on Windows). The script automatically downloads the model file if it does not yet exist, and then runs the executable.

macOS / Linux

./gemma4.sh

Windows

gemma4.bat

If you want to download the model file manually, you can obtain it from the URL below and place it in the same folder as the executable.

https://storage.googleapis.com/ailia-models/gemma/gemma-4-E2B-it-Q4_K_M.gguf

Then run the executable directly:

./gemma4 -m gemma-4-E2B-it-Q4_K_M.gguf

Platform-specific remarks

Release the 'downloaded' attribute in MacOS

Binaries downloaded from the browser in macOS may have a 'downloaded' attribute and may not be able to be executed. In that case, it is possible to remove the attribute by right-clicking on the .dylib file and executing it. You can also release the 'downloaded' attribute by executing the following command from the command line:

xattr -d com.apple.quarantine libailia_llm.dylib
